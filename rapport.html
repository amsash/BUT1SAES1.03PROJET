Installation d’un poste pour le développement — Gr2 Binome102

Rapport

1. Introduction

L'objectif de ce SAE est de comprendre et d'utiliser les technologies de virtualisation afin d'installer un poste de travail de développement.

Le projet consiste à créer une machine virtuelle, à installer un système d'exploitation Linux, à configurer des comptes utilisateurs, à mettre en place un serveur web et à publier le rapport de projet sur ce serveur.

Les outils utilisés dans ce projet sont les suivants :

  • VirtualBox pour la virtualisation
  • Ubuntu Linux comme système d'exploitation
  • Apache comme serveur web
  • WebStorm pour le développement et le déploiement web (avec l'aide de GitHub)

2. Concepts de virtualisation

Définition de la virtualisation

La virtualisation est une technologie qui permet à plusieurs systèmes d'exploitation de fonctionner sur la même machine physique en partageant des ressources matérielles telles que le processeur, la mémoire et le stockage.

Émulateur, simulateur et machine virtuelle

  • Émulateur : reproduit à la fois le matériel et les logiciels d'un autre système (lent, utilisé pour des raisons de compatibilité).
  • Simulateur : simule un comportement à des fins de test ou de formation.
  • Machine virtuelle : exécute un système d'exploitation invité à l'aide du matériel de l'hôte via un hyperviseur.

Type de virtualisation utilisé

Dans ce projet, un hyperviseur hébergé (type 2) a été utilisé.

VirtualBox s'exécute au-dessus du système d'exploitation hôte et gère la machine virtuelle.

3. Configuration de la machine virtuelle

Environnement hôte

  • Système d'exploitation hôte : Windows 11
  • Logiciel de virtualisation : VirtualBox

VirtualBox a été choisi car il est gratuit, facile à utiliser et prend en charge les instantanés et les dossiers partagés.

Configuration de la machine virtuelle

La machine virtuelle a été configurée avec les paramètres suivants :

  • Système d'exploitation : Ubuntu 22.04 LTS
  • RAM: 4 Go
  • Processeur : 1 cœur
  • Storage: 35 Go

Ces valeurs ont été choisies afin de garantir de bonnes performances tout en maintenant une utilisation raisonnable des ressources.

4. Configuration réseau

Choix du mode réseau

La machine virtuelle utilise NAT avec redirection de port.

Ce choix a été fait car l'ordinateur hôte est connecté à un réseau Wi-Fi partagé dans un appartement, qui peut bloquer la communication directe entre les appareils.

Redirection de port

La redirection de port a été configurée comme suit :

  • Port hôte : 8080
  • Port invité : 80

Cela permet d'accéder au serveur web depuis la machine hôte à l'aide de :

http://localhost:8080

5. Gestion du système Linux et des utilisateurs

Choix du système d'exploitation

Ubuntu Linux a été choisi pour sa stabilité, sa sécurité et le soutien important de sa communauté.

Création d'utilisateurs et de groupes

Les utilisateurs et groupes suivants ont été créés :

  • Administrateurs: admin_gr2_binome102
  • Utilisateurs: binome102_1, binome102_2
  • Groupe: binome102

Les noms d'utilisateur en minuscules ont été utilisés afin de respecter les conventions de nommage Linux.

6. Installation et configuration du serveur Web

Installation d'Apache

Le serveur web Apache a été installé à l'aide du gestionnaire de paquets du système.

Le service a été activé et démarré automatiquement.

Configuration de l'hôte virtuel

Un hôte virtuel dédié a été créé pour héberger le site web du projet.

La racine du document pointe vers le répertoire du projet contenant les fichiers HTML.

accès HTTP

Le site web est accessible via HTTP sur le port 80.

7. Développement et déploiement de sites Web

Technologies utilisées

  • HTML5 avec balises sémantiques
  • Bootstrap 5 pour la mise en page et la réactivité

Méthode de déploiement

Un dépôt GitHub a été utilisé pour partager le code de balisage du site Web entre la machine hôte et la machine virtuelle.

Cette méthode a été choisie pour sa simplicité et sa fiabilité.

8. Conclusion

Ce projet a permis de comprendre le fonctionnement de la virtualisation et la configuration d'un environnement de développement complet.

Des compétences pratiques ont été acquises en administration de systèmes Linux, en mise en réseau et en configuration de serveurs web.

Ce SAE a fourni une introduction solide aux scénarios réels de déploiement et de dépannage.
